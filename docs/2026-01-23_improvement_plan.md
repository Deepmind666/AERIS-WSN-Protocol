## 0. 总原则
1. **所有结论必须能回溯到 JSON / 脚本 / 数据路径**。
2. **不使用未验证文献**，任何不确定的引用必须 `TODO` 标记。

5. **审稿视角**：优先解决会被抓住的漏洞（数据一致性、对比完整、统计严谨）。

## 2. 中期改进任务（P1）

### P1‑1 多数据集对照
- 目的：避免“只用 Intel 数据集”导致泛化质疑。
- 动作：
  - 补充 1–2 个公开 WSN 相关数据集（或模拟场景族）。

### P1‑2 消融实验扩展
- 目的：证明 AERIS 内部模块的贡献。
- 动作：
  - “无 CAS / 无 Skeleton / 无 Gateway / 无 Safety”的系统级消融。
  - 统一统计方法（CI + Holm‑Bonferroni）。

### P1‑3 统计报告精简
- 目的：避免正文被统计表淹没。
- 动作：
  - 主文只保留核心 p‑value/效应量结论。
  - 详细统计表全部移至 Supplementary。
